# testTaskRepo
Репозиторий, куда собраны наиболее интересные мне задачи. Цель: переиспользовать данный материал для подготовки intern\junior специалистов имея актуальные задачи и решения на них, также самостоятельно вспомнить аспекты языка.

### Краткая идеология данного репозитория
В репозитории содержатся задачи уровня junior\middle, задачи уровня senior, относятся к архитектурных подходам, которые будут включены отдельными категориями
_____________
Исходный материал сгенерирован моделями Deepseek-v3.2\\gpt4.5\\glm4.7, ревью проводилось этими же моделями, где подход был применим, сгенерированная задача моделью А, проверялась моделью Б в случайном порядке.
_____________
В данном репозитории могут быть неидеальные решения, поскольку был наложен авторский (мой) отпечаток и некрасивые или непонятные решения нейросети, были заменены более простыми и удобочитаемыми вариантами.

🔹 Пул 1. Дополнительное закрепление (по существующим темам)
Задачи опираются на те же концепции, что уже есть в репозитории, но с новыми условиями и небольшим усложнением.

1.1 Калькулятор с историей (тема: «Простой калькулятор»)
Условие
Модифицируйте консольный калькулятор так, чтобы он запоминал последние 5 операций (каждая операция – это строка вида "2 + 3 = 5"). Добавьте команду history для вывода всех сохранённых операций. Если операций меньше пяти – выводить все.

1.2 Палиндром с учётом знаков препинания (тема: «Проверка палиндрома»)
Условие
Напишите метод isPalindromeExtended(String s), который возвращает true, если строка является палиндромом, игнорируя пробелы, знаки препинания и регистр.
Пример: "А роза упала на лапу Азора" → true.
"Madam, I'm Adam" → true.
"Это не палиндром!" → false.

1.3 Поиск двух максимальных элементов (тема: «Поиск max в массиве»)
Условие
Дан массив целых чисел (не отсортирован). Найдите два наибольших различных элемента за один проход по массиву (без сортировки). Если все элементы одинаковые – вернуть этот элемент и null.
Пример: [3, 7, 2, 7, 5] → (7, 5).
[4,4,4,4] → (4, null).

1.4 Stream API: фильтрация и сумма (тема: «Stream API: квадраты чисел»)
Условие
Дан список целых чисел. С помощью Stream API:
отфильтровать только положительные числа, затем преобразовать их в квадраты, вычислить сумму этих квадратов.
Напишите метод sumOfPositiveSquares(List<Integer> numbers).

1.5 Сравнение объектов: клонирование (тема: «Сравнение объектов»)
Условие
Добавьте в класс Person методы: clone() (поверхностное копирование), конструктор копирования Person(Person other).
Напишите тест (в main или JUnit), который показывает разницу между ссылкой на тот же объект и независимой копией.

1.6 Потокобезопасный счётчик с ограничением (тема: «ConcurrentCounter»)
Условие
Реализуйте BoundedCounter, который: хранит значение от 0 до max (задаётся в конструкторе), методы increment() и decrement() – потокобезопасны (используйте AtomicInteger или synchronized), при попытке увеличить выше max – бросает IllegalStateException, при попытке уменьшить ниже 0 – тоже исключение.

1.7 Try-with-resources: работа с файлом (тема: «Try-with-resources»)
Условие
Напишите метод copyFile(String src, String dst), который копирует содержимое одного текстового файла в другой, обязательно используя try-with-resources для BufferedReader и BufferedWriter. Если исходный файл не существует – выбросить FileNotFoundException с понятным сообщением.

🔹 Пул 2. Расширенные задачи (новые темы, не покрытые в репозитории)
Эти задачи добавят новые разделы: работа со временем, Optional, собственные исключения, коллекции (Map/Sort), JDBC (лёгкий), алгоритмы поиска.

2.1 Работа с датами: валидация возраста (тема: java.time)
Условие
Напишите класс AgeValidator с методом boolean isAdult(LocalDate birthDate), который проверяет, что возраст на текущую дату составляет 18 лет или больше. Используйте LocalDate.now() и Period. Добавьте метод String getAgeDescription(LocalDate birthDate), который возвращает строку "Вам X лет, Y месяцев, Z дней".

2.2 Optional вместо null (тема: Optional)
Условие
Дан метод User findUserById(int id), который может вернуть null. Перепишите его в стиле Optional<User> findUserOptional(int id). Напишите метод String getCityOfUser(int id), который возвращает город пользователя или строку "Unknown", используя Optional (без if и isPresent() – только map/orElse).

2.3 Собственное checked исключение (тема: исключения)
Условие
Создайте класс InsufficientFundsException extends Exception. Напишите класс BankAccount с полями balance и методами withdraw(double amount) – если сумма больше баланса, бросайте InsufficientFundsException. В main перехватите исключение и выведите понятное сообщение.

2.4 Подсчёт частоты слов (тема: Map, HashMap)
Условие
Напишите метод Map<String, Integer> wordFrequency(List<String> words), который возвращает карту: слово → сколько раз встречается. Используйте HashMap. Дополнительно: метод void printSortedByFrequency(Map<String, Integer> freq), который выводит слова в порядке убывания частоты (при равной частоте – по алфавиту). Подсказка: используйте Collections.sort с кастомным компаратором.

2.5 Алгоритм: бинарный поиск (тема: алгоритмы)
Условие
Реализуйте метод int binarySearch(int[] sortedArray, int key), который возвращает индекс элемента или -1, если не найден. Не используйте встроенный Arrays.binarySearch. Напишите также рекурсивную версию binarySearchRecursive.

2.6 Простой чат с использованием BlockingQueue (тема: многопоточность)
Условие
Создайте два потока: писатель (раз в 1 секунду отправляет строку в очередь) и читатель (забирает и выводит). Используйте LinkedBlockingQueue<String>. Остановите работу через 5 секунд.

2.7 JDBC: работа с H2 (тема: базы данных)
Условие
Подключите H2 (in-memory). Создайте таблицу products(id INT PRIMARY KEY, name VARCHAR(100), price DECIMAL). Напишите методы:

insertProduct(Product p)

List<Product> getProductsCheaperThan(double maxPrice)
Используйте PreparedStatement и try-with-resources для Connection, Statement.
